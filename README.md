# picgo-plugin-autobackup

## 简介

将图片存储在免费的`公共图床`是非常吸引人的，尤其是那些带有`cdn`加速的图床。然而，这些图床的稳定性总让我们担忧，如何既能让我们享受公共图床带来的便利，又能解决后顾之忧？也许将图片备份下来是个不错的选择！当公共图床失效时，我们可以重新上传这些图片，而不必担心永远失去它们，本插件就是一款专注于图片备份的插件。

<br>

## 插件特点

- 支持开启/关闭插件

- 支持自定义备份位置 (目前支持本地备份，未来将支持更多，欢迎更多开发者加入! )
- 支持自定义备份文件夹

<br>

## 插件前瞻

该插件会将上传图片名称、链接、上传时间等信息记录下来，以便后期重新上传。未来将开发更多功能 (但可能独立于该插件)：

- 备份图片至坚果云、比邻云
- 重新上传并自动修改文章中的链接
- 同步备份至多平台
- ……

<br>

## 环境搭建

1. `GUI`用户直接在`插件设置`中搜索`autobackup`下载安装

   ![image-20220215222022205](https://img1.imgtp.com/2022/02/15/kApL4Y22.png)

<br>

2. 上传一张照片 (上传照片后插件会自动在picgo配置文件中创建插件配置内容)

3. 根据个人喜好修改设置，`windows`配置文件路径为`C:\Users\username\AppData\Roaming\picgo\data.json`

![image-20220215222537485](https://img1.imgtp.com/2022/02/15/ccBQShVH.png)

- `enable`：是否启用该插件，`true`为启动，`false`为关闭

- `markFilePath`：mark.json文件存储路径

- `imagePath`：备份图片存储文件夹路径

- `backupLocation`：图片备份位置 (local-本地)

  初次安装插件并上传图片后，若您没有修改`picgo`设置文件，软件会自动在`Picgo`安装目录下新建`mark.json`和`Image文件夹`，其分别用于存储`图片信息`和`备份图片`。请注意：

  1. 若您修改了`markFilePath`，则您需要将`picgo`安装目录下的`mark.json`复制到新的路径下，或者手动在新路径下创建`mark.json`

     ```json
     {
         "images": []
     }
     ```

  2. 若您修改了`imagePath`，则您需要手动创建该文件夹

<br>

4. 设置完成！
